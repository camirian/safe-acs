# Mission Engineering & Architecture (UAF)

This directory contains the operational and systemic architectural views for the **SafeACS** (Cyber-Physical AI Assurance Framework) mission.

## DoD Mission Engineering Guide (MEG) Context
In alignment with the principles outlined in the **DoD Mission Engineering Guide**, SafeACS evaluates the "system" not merely as a piece of isolated software, but as an **End-to-End Mission Capability**. 

The core mission of SafeACS is: *To ensure the secure, un-poisoned, and deterministic operation of critical Space Vehicle (SV) hardware while leveraging probabilistic edge AI.*

By mapping our solutions via the Unified Architecture Framework (UAF), we provide traceable, mission-level justification for why the Bimodal Decision Protocol (the physical edge code) is structurally necessary to secure the mission objectives against the unique threat vectors of adversarial LLMs.

## Architectural Views (UAF / DoDAF)

The following views define the operational context and activity flows of the mission:

- **[UAF Op-Co (OV-1): High-Level Operational Concept](UAF_Op-Co_High_Level_Concept.md)**: Illustrates the macro mission environment, the participating nodes, and the security boundaries.
- **UAF Op-Pr (OV-5b): Operational Processes** *(In Development)*: Details the exact sequence of activities and data exchanges between the Space Segment, Edge Node, and Cloud Logic.
